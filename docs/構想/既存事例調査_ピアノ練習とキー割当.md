# 既存事例調査：ピアノ練習とキー割当

## 調査目的

PCキーボードを鍵盤に見立てる設計の参考として，既存サービスの機能と制約を整理する．

## 既存事例

1. Virtual Piano

- URL: https://virtualpiano.net/
- 特徴: QWERTYキーで演奏可能．
- 補足: キーボード入力前提の練習導線があるため，キー割当設計の参考になる．

2. Recursive Arts Virtual Piano

- URL: https://recursivearts.com/virtual-piano/
- 特徴: パソコンキーボードでの演奏を明示．
- 補足: 鍵盤UIとQWERTY対応の併用例として参考になる．

3. Synthesia

- URL: https://synthesiagame.com/
- 特徴: 落下ノーツ型．MIDIキーボード連携を重視．
- 補足: 本格練習は物理鍵盤前提である点が示唆になる．

4. Piano From Above

- URL: https://pianofromabove.en.softonic.com/
- 特徴: MIDIを使った落下ノーツ練習ソフトの系譜．
- 補足: ノーツ可視化や速度調整の考え方を取り入れられる．

## キー割当方針の候補

- 候補A: 2オクターブを英字キー中心で固定割当．
- 候補B: 白鍵モード（黒鍵自動補完）とフルモードを切替．
- 候補C: 曲に必要な音だけを動的割当．

## 本プロジェクトへの適用方針（暫定）

- 初期実装は候補Aで開始し，実装複雑度を抑える．
- 学習モードで候補Bを導入し，初心者の負荷を下げる．
- 将来的に候補Cを追加し，難曲対応を進める．
